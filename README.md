# Bigin 과제 프로젝트

### 객체지향 설계 원칙을 최대한 지키기 위해 노력
- 개방 폐쇄 원칙
   Weapon인터페이스를 만들어서 Character는 Weapon 인터페이스를 의존하게 만들어 확장성을 높이게 만들었습니다.

- 인터페이스 분리 원칙
   Character클래스와 Monster클래스에 많은 메서드를 의존하고 있어 책임이 너무 많아 인터페이스 분리 원칙을 사용했습니다.
    특히 BasicSkillRemoveThread에서 Character의 인터페이스를 의존하게 된다면 사용하지 않는 많은 메소드에 책임을 갖게 되었습니다.
    따라서 CharacterSkill인터페이스를 만들어서 BasicSkillRemoveThread는 CharacterSkill를 의존하게 만들고 메서드의 책임을 줄이면서 원하는 메서드만
    책임을 갖게 했습니다.

- 단일 책임 원칙
     각 클래스는 한 가지 책임만 가질 수 있도록 추상화 시켰습니다.

- 리스코프 원칙
     하위 클래스는 상위 클래스이다 라는 원칙을 생각하면서 엘프, 휴먼, 오크는 캐릭터라는 생각을 하며 리스코프 원칙을 지키도록 노력했습니다.

- 의존관계 역전 원칙
     위에서 말한 개방 폐쇄 원칙을 지키니 Character에서 구체적인 Weapon의 무기 클래스를 의존하지 않고 Weapon인터페이스를 의존하게 하여 의존 역전 법칙을 지키게 됐습니다.   

### 전략 패턴 
- Character추상 클래스의 wear메소드에서 전략 패턴 사용
